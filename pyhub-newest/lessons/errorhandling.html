Introduction

Error handling is the process of catching and managing runtime errors in a program to prevent it from crashing. Python uses exceptions to represent errors, and you can handle them using try, except, finally, or even raise your own exceptions.

Try and Except

try:
    x = 5 / 0  # This will cause an error
except ZeroDivisionError:
    print("Cannot divide by zero!")

Output:

Cannot divide by zero!

Finally

try:
    file = open("myfile.txt", "r")
    content = file.read()
    print("File content:")
    print(content)
except FileNotFoundError:
    print("Oops! myfile.txt doesn't exist yet.")
finally:
    print("Operation complete.")

Output:

Oops! myfile.txt doesn't exist yet.
Operation complete.

Raising Exceptions

Raising exceptions means creating your own errors when something unexpected happens in your code.

def check_age(age):
    if age < 0:
        raise ValueError("Age cannot be negative!")
    elif age < 18:
        raise ValueError("You must be at least 18 years old.")
    else:
        print("Access granted!")

# Test the function
try:
    check_age(-5)
except ValueError as e:
    print(f"Error: {e}")

Output:

Error: Age cannot be negative!
